# Link Component

The Link component is a versatile navigation element that seamlessly integrates with Next.js routing while providing multiple visual variants and extensive customization options. It automatically handles internal and external links with proper security attributes and accessibility features.

- üéÆ **Gaming Themed** - Special neon and gold theme support

## Basic Usage

```tsx
import { Link } from '@/components/ui/Link';

// Internal navigation
<Link href="/games">Browse Games</Link>

// External link
<Link href="https://example.com" external>
  Visit External Site
</Link>

// With icons
<Link 
  href="/dashboard" 
  leftIcon={<Home />}
  variant="primary"
>
  Dashboard
</Link>
```

### Icon Guidelines

- Use **16px** icons for small/medium links
- Use **20px** icons for large/xl links
- Icon-only links should include `aria-label`
- External links auto-add indicator unless custom `rightIcon` is provided

## Accessibility

The Link component follows WCAG guidelines:

- **Keyboard Navigation** - Full keyboard support with focus indicators
- **Screen Readers** - Proper ARIA labels and announcements
- **Touch Targets** - Minimum 44x44px touch targets on mobile
- **Color Contrast** - AAA compliant contrast ratios
- **Focus Indicators** - Clear visible focus states

```tsx
// Accessible icon-only link
<Link 
  href="/settings"
  iconOnly
  aria-label="Open settings"
>
  <Settings />
</Link>

// Accessible external link
<Link 
  href="https://example.com"
  external
  aria-describedby="external-site-info"
>
  Visit Partner Site
</Link>
```

## Best Practices

### Do's ‚úÖ

- Use semantic variants that match the link purpose
- Include descriptive text or aria-labels
- Use `external` prop for external links
- Choose appropriate sizes for context
- Test keyboard navigation

### Don'ts ‚ùå

- Don't use vague text like "Click here"
- Don't override Next.js routing unnecessarily
- Don't use buttons for navigation (use Link)
- Don't forget aria-labels on icon-only links
- Don't disable links without clear indication

## Migration Guide

### From HTML Anchor

```tsx
// Before
<a href="/games" className="custom-link">Games</a>

// After
<Link href="/games" variant="primary">Games</Link>
```

### From Next.js Link

```tsx
// Before
import Link from 'next/link';
<Link href="/games">
  <a className="link-styles">Games</a>
</Link>

// After
import { Link } from '@/components/ui/Link';
<Link href="/games" variant="primary">Games</Link>
```

### From Button with onClick Navigation

```tsx
// Before
<Button onClick={() => router.push('/games')}>
  Browse Games
</Button>

// After
<Link href="/games" variant="primary">
  Browse Games
</Link>
```
