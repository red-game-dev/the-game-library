# ProgressBar

A versatile progress indicator component with multiple variants, animations, and customization options. Perfect for showing loading states, file uploads, downloads, or any process completion.

<Canvas of={ProgressBarStories.Default} />

## Features

- **Multiple Variants**: Default, primary, secondary, success, warning, error, and gradient
- **Responsive Sizes**: Extra small to extra large with mobile-first design
- **Animations**: Striped, animated stripes, shine, pulse, and loading effects
- **Customizable**: Labels, colors, rounded corners, and glow effects
- **Theme Support**: Works perfectly with all themes (light, dark, neon, gold)
- **Accessibility**: ARIA attributes and keyboard navigation support

## Usage

```tsx
import { ProgressBar } from '@/components/ui/ProgressBar';

// Basic usage
<ProgressBar value={60} />

// With label
<ProgressBar value={75} showLabel />

// Custom variant and size
<ProgressBar value={80} variant="success" size="lg" showLabel />

// Animated with effects
<ProgressBar 
  value={70} 
  variant="gradient"
  striped
  animated
  glow
  shine
/>
```

- Support for reduced motion preferences

## Best Practices

1. **Use appropriate variants**: Match the variant to the context (success for completion, error for failures)
2. **Show labels for clarity**: Include percentage or status text when the exact progress matters
3. **Choose the right size**: Use smaller sizes in compact spaces, larger sizes for emphasis
4. **Consider animation**: Use animated stripes for active processes, static for paused states
5. **Provide context**: Always include descriptive text near the progress bar explaining what's happening
6. **Handle edge cases**: Account for 0% and 100% states appropriately
