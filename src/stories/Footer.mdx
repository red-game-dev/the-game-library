# Footer

A comprehensive site footer with theme switcher, navigation links, social media, and company information. The Footer provides essential site navigation and information while maintaining a clean, organized layout.

<Canvas of={FooterStories.Default} />

## Features

- **Theme Switcher**: Quick access to switch between light, dark, neon, and gold themes
- **Navigation Links**: Organized sections for Games, Features, Support, and Company
- **Social Media Links**: Connect with social platforms
- **Newsletter Signup**: Optional email subscription form (can be toggled)
- **Responsive Design**: Mobile-first approach with optimized layouts
- **Coming Soon Badges**: Automatic detection and display of upcoming features
- **Legal Links**: Terms, Privacy, and Cookie policies
- **Play Safe Notice**: Gaming responsibility reminder

## Usage

```tsx
import { Footer } from '@/components/layout/Footer';

// Basic usage
<Footer />

// Without newsletter
<Footer showNewsletter={false} />

// Compact version
<Footer compact={true} />

// Custom configuration
<Footer 
  showThemeSwitcher={true}
  showSocials={true}
  showNewsletter={false}
  compact={false}
/>
```

The Footer uses a smart badge system that:

## Best Practices

1. **Newsletter Control**: Enable newsletter only when the feature is ready
2. **Social Links**: Keep social media links updated and valid
3. **Route Configuration**: Use centralized route constants for consistency
4. **Theme Persistence**: Theme selection should be saved to localStorage
5. **Responsive Testing**: Always test footer on mobile devices
6. **Legal Compliance**: Ensure all legal links are present and functional

## Layout Integration

When integrating the Footer in your layout:

```tsx
// In your layout component
<div className="app-layout">
  <Header />
  <main className="app-layout-main">
    {children}
  </main>
  <Footer 
    showNewsletter={false}  // Enable when ready
    showThemeSwitcher={true}
    showSocials={true}
  />
</div>
```

The layout uses flexbox to ensure the footer stays at the bottom:

```css
.app-layout {
  min-height: 100vh;
  display: flex;
  flex-direction: column;
}

.app-layout-main {
  flex: 1;
}
```

## Configuration

Footer links and badges are managed through centralized configuration:

```tsx
// Routes are defined in routes.constants.tsx
const config = getRouteConfig(path);
if (config?.comingSoon) {
  // Automatically shows "SOON" badge
}

// Social links from constants
SOCIAL_LINKS.TWITTER
SOCIAL_LINKS.FACEBOOK
// etc.
```

## Mobile Optimization

On mobile devices:
- Navigation sections stack vertically
- Theme buttons are icon-only to save space
- Social links are displayed in a compact row
- Legal links are centered
- Newsletter form stacks vertically

## Performance Considerations

- Theme switcher updates are optimized using Zustand store
- Route configurations are memoized
- Social links use external link attributes
- Lazy loading for non-critical sections when needed
