# The Game Library Design System

A comprehensive custom CSS design system built from scratch, providing Tailwind-like utility classes without framework dependencies.
Optimized for gaming platforms with theme switching, accessibility, and performance in mind.

## ğŸ¯ Design Philosophy

### Why Custom CSS Over Tailwind

We chose to build a custom design system instead of using Tailwind CSS v4:

1. **Framework Independence** - No build tool complications or configuration issues
2. **Tailwind v4 Alpha Issues** - Avoided 4+ hours of debugging unstable APIs
3. **Complete Control** - Full ownership of design tokens and utilities
4. **Future-Proof** - Can add Tailwind later without breaking changes
5. **Performance** - Lighter bundle without unused utilities

### Core Principles

1. **Gaming-First Aesthetics**
   - Electric purple for Web3 identity
   - Cyber lime for success and wins
   - Neon glows and glass morphism effects
   - Dark theme as default

2. **Accessibility First**
   - WCAG AAA contrast targets
   - Clear focus states
   - Semantic HTML structure
   - Screen reader optimized

3. **CSS Variables Architecture**
   - Theme switching via CSS custom properties
   - Runtime theme changes without recompilation
   - Consistent design tokens across all utilities

## ğŸŒ— Theme System

### Available Themes

1. **Dark (Default)**
   ```css
   :root {
     --color-background: #030712;
     --color-surface: #0f172a;
     --color-text: #f8fafc;
   }
   ```

2. **Light**
   ```css
   :root[data-theme="light"] {
     --color-background: #ffffff;
     --color-surface: #f8fafc;
     --color-text: #030712;
   }
   ```

3. **Neon**
   ```css
   [data-theme="neon"] {
     --color-primary: var(--cyan-400);
     --color-accent: var(--pink-500);
   }
   ```

4. **Gold**
   ```css
   [data-theme="gold"] {
     --color-primary: var(--yellow-400);
     --color-accent: var(--orange-500);
   }
   ```

### Theme Switching

```tsx
const handleThemeChange = (theme: string) => {
  document.documentElement.setAttribute('data-theme', theme);
};
```

## ğŸ”„ Migration Path

Our custom system is designed to work alongside Tailwind if needed:

1. **Compatible Naming** - Uses Tailwind conventions (bg-*, text-*, etc.)
2. **CSS Variables** - Can coexist with Tailwind's design tokens
3. **No Breaking Changes** - Add Tailwind later without refactoring
4. **Gradual Migration** - Both systems can work together
